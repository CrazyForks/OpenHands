# Environment Setup — openhands

Writes `config.template.toml` from OpenHands secrets before run commands.

## Detected Config

- **Config file:** `config.template.toml`
- **Required vars:** `LLM_API_KEY, GITHUB_TOKEN`
- **Optional vars:** `DATABASE_URL, REDIS_URL`

## Process

### Step 1: Check for Existing Config Template

```bash
# Check if .env.example or similar template exists
ls -la .env* 2>/dev/null
cat .env.example 2>/dev/null || cat .env.sample 2>/dev/null || cat config.template.toml 2>/dev/null || echo "No template found"
```

### Step 2: Read OpenHands Secrets

```bash
# OpenHands injects secrets as environment variables
# Available secrets should be listed in OpenHands settings
# For this template, we use common patterns:
echo "Checking for common secrets..."
env | grep -E "API_KEY|TOKEN|SECRET|DB_|DATABASE|PASSWORD" | sed 's/=.*/=***/' || echo "No secrets found in environment"
```

### Step 3: Write Config File

```bash
# Create config.toml file with secrets
cat > config.toml << 'EOF'
# Environment Configuration
# Generated by pipeline-generator automation

# LLM Configuration
LLM_API_KEY=${LLM_API_KEY}
LLM_MODEL=gpt-4o

# GitHub Integration
GITHUB_TOKEN=${GITHUB_TOKEN}

# Database (if needed)
# DATABASE_URL=${DATABASE_URL}

# Redis (if needed)
# REDIS_URL=${REDIS_URL}

# Application Settings
DEBUG=false
LOG_LEVEL=info
EOF

echo "Config file created"
```

### Step 4: Verify Config File

```bash
# Ensure config file was created
ls -la config.toml 2>/dev/null || ls -la config.toml.template 2>/dev/null || echo "Config file not found"
```

## Common Patterns

### Python (pyproject.toml based)
```
# config.toml or .env
DATABASE_URL=postgresql://user:pass@localhost/db
SECRET_KEY=your_secret_key_here
DEBUG=True
```

### Node.js (package.json based)
```
# .env
DB_CONNECTION=mongodb+srv://user:pass@cluster.mongodb.net/db
API_KEY=your_api_key_here
PORT=3000
NODE_ENV=development
```

## Error Handling

If config file cannot be written:
1. Log warning to `/tmp/env-setup-warnings.log`
2. Continue with pipeline (some apps have default configs)
3. Document missing config in PR comment
